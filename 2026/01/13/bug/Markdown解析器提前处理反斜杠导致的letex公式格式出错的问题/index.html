Markdown解析器提前处理反斜杠导致的letex公式格式出错的问题

Created2026-01-13|Updated2026-01-13|bug
|Post Views:

问题

在hexo中，默认的Markdown渲染器会转义或删除多余的反斜杠
而Latex中的“\”表示换行，但在Maekdown中会被当作“字面反斜杠”处理导致“\”被转义称\或直接消失

解决方案

对每个\进行双重转义，将\写成\\
原理：Markdown先把\\转义成\再交给Mathjax，Mathjax就能正确识别为换行
或者安装pandoc渲染器：
npm uninstall hexo-renderer-marked
npm install hexo-renderer-pandoc
这样就可以正常写
